第1节：初始数据结构与算法

2025-03-04 数据结构与算法

  如果数据结构是食材和条例，那么算法就是烹饪方式。厨师需要选择合适的食材与合适的烹饪方式。而我们需要选择合适的数据结构与算法。

1.1 数据结构

  数据结构就是数据的组织结构，用来组织，存储数据。数据结构有两种分类方式，分别是按逻辑结构分类和按物理结构分类。

  数据的逻辑结构分为集合结构、线性结构、树形结构和图形结构。

  1. 集合结构：与数学中集合的定义类似，其中的数据元素没有顺序，并且每个数据元素都是唯一的。
  2. 线性结构：数据元素排成一排，除了首和尾，每个数据元素的前面和后面都有一个相邻的数据元素。
  3. 树形结构：数据元素之间是一对多的层次关系。
  4. 图形结构：数据元素之间是多对多的关系。

  数据物理结构有很多种，采用最多的是顺序存储结构和链式存储结构。

  1. 顺序存储结构：将数据存放在一片地址连续的存储单元中，数据元素之间的逻辑关系通过数据元素的存储地址来直接反应。在这种数据结构中，逻辑上相邻的数据元素在物理地址上也必然相邻。
    优点：简单、易理解，且实际占用空间最小。
    缺点：需要占用一片连续的存储单元且存储分配要事先进行，另外对于一些操作的时间效率较低（移动、删除元素等操作）
  2. 链式存储结构：将数据元素随机地存储在物理单元中，以链结点的形式储存，不仅存放一个数据元素的数据信息，还存放这个元素在逻辑关系上的直接后继元素所在链结点的地址，该地址被称为指针。换句话说，数据元素之间的逻辑关系是通过指针来间接反映的。
    优点：存储空间不必事先分配，可以在需要时临时申请，更节约空间。插入，移动，删除等操作的时间效率远比顺序存储结构高
    缺点：空间开销更大，不仅要存储数据信息，还要存放指针。

1.2 算法

  算法是一系列运算步骤，表达解决某一类问题的一般方法，对这类问题的任何一个输入，可以通过这个方法最终产生一个输出。
算法的基本特性如下所示：

  1. 输入
  2. 输出
  3. 有穷性：算法必须在有限步内结束，且应该在一个可接受的时间内完成。
  4. 确定性：组成算法的每一条指令都必须有明确的含义。
  5. 可行性：每一步操作都必须具有可执行性。

算法追求的目标是：最低的时间复杂度和空间复杂度
